# Pixboom 搜人筛人作业流程
> ⚠️ **改本 Skill 前先想**:这里只放**所有岗通用、可复用的作业流程**。单一案例/某轮/某岗经验 → 进该岗 `JD.md`,不进这里。
> 每岗四件:`JD.md`(对外 JD + 内部备注表,**判断只看它**)· `去重台账.csv`(`resume_id,status`,仅去重)· `候选人池.md`(确认候选人)· `待定/`(简历 PDF 中转)。
> **路径**:命令里 `<岗位>\…` 相对工作区根(本 `CLAUDE.md` 所在);脚本都在 `.claude/skills/pxb-liepin/scripts/`(下文简写 `SC`)。机械活全在脚本、跨 Win/mac;判断在本文 + agent。

## 全流程:两层循环,主 agent 串行自跑
**P. 聊全 JD** → 0. 抢锁 → 0.5 登录 → 1. 粗筛 → 2. 精筛 → 3. 够一批没(没够回 1)→ 4. 出 PDF 交用户 → 5. 入池 → 6. 迭代回 0。

- **不派 subagent**:单账号单锁串行,subagent 无并行收益、反而启动慢易卡死;主 agent 从头跑到尾。
- **小循环(1-3,自跑、不打扰用户)**:搜一页 → 粗筛挑 ~10 → 精筛 → 对口标 `待定` → 没攒够回步骤 1,连轮跑到攒够 ~5 个 `待定`。**绝不跑一轮就停等用户**。
- **大循环(4-6,用户校准 JD)**:出一批 PDF → 用户复核 → 入池 → 迭代回写 JD(**重点是进化关键词**)→ 下一批。JD 越调越准、小循环越搜越准。
- **上下文**:脚本只输出窄表。看完做完判断后**记住结论即可**(`待定` 的 id=标签、方向感、哪个词有效),别把整页窄表一直留着;轮数多了丢旧轮明细。

### 状态机(去重台账只此 5 值,全程走 `dedup.mjs`,绝不手改 CSV)
`未精筛`(捞回来没细看的**干净库存**)→ `粗筛不合适`(卡片就否)/`精筛不合适`(细看后否,含用户毙掉的)/`待定`(细看够格、出了 PDF 等复核)→ `入选`(用户拍板要了)。

### 心跳 + 输出纪律(治"闷头静默"和"过程废话/散问打断")
- **边跑边报**:每跑完一轮先发**一行**进度再继续,别让用户 10-20 分钟看不到动静。格式:`第N轮(词:X):搜A/精筛B/新增待定C,累计 D/目标 E,继续…`。心跳是单向,不算打扰。
- **心跳就那一行**:逐人淘汰理由、本轮发现、推理**全留你脑子里 + JD 表,别讲给用户**。
- **交付干净**:出批消息 = 几份可点 PDF + 问为什么,别夹每轮过程、复盘、SKILL 自测。
- **决策别打断、给推荐、能不问就别问**:破例/放宽约束/停搜这类才需用户拍的,优先**带推荐先做可逆一步**(照推荐先提报、复核可撤),决策点在交付里一句话点出;真要问就**攒一处、一次问清、每条附推荐**,别过程里东一句西一句、也别没问完就干等。照 JD 能自己判的,别问。

### 全程红线
① 单账号串行,**绝不并发**调 liepin。② 反爬宁慢勿封:脚本非零退出即停、别密集连刷(脚本内已带抖动)。③ 原始 JSON 不进上下文(脚本只输出窄表)。④ 隐私:简历只存本地、不外发、不传外部服务。⑤ 只 `search`/`resume`,**不 `greet`**(沟通归 HR)。⑥ **只走脚本**(`search`/`fetch`/`pdf`/`jd-round`,已强制无头),除用户手动 `liepin login` 外**绝不裸敲 `liepin`**(裸调发布版默认有头会弹窗)。全程**不该弹任何浏览器窗口**;弹了 = 有人裸调或没登录 → **停手排查、别继续刷**(可选安全网:装完跑一次 `patch-headless.mjs`)。

> 脚本无参运行会打印用法;任一脚本退出码 ≠ 0 = 停手,按报错处置(多半反爬/被踢/未登录)。

## P. 开搜前:把 JD + 人员画像聊全(主 agent+用户,**不碰 liepin、不抢锁**)
搜任何人**之前**,先把该岗 **JD + 人员画像**补到能搜的状态——**判断全程只看它,补不全 = 瞎搜、多轮返工**。在对话里完成。

> 🟢 **怎么聊(最重要)**:像懂行的猎头跟用人方对齐需求,**用人话**把"要什么样的人、这种人在哪、什么样的看着像其实不是"聊明白。下面的"三锚/锚一二三"是**给你对照的内部清单,不是台词**——别把这些词、别"锚一锚二"这样列条目念给客户。自然地问、一次一两点。

**① 补事实(填 JD 硬约束行)**:预算/城市/经验/学历/年龄/目标人数。缺的顺嘴问,**别追问"预算够不够"**——按预算搜,真枯竭(步骤3)再谈。
**② 聊画像(内部对照"三锚";别问"最看重什么能力/不要哪类人"这种抽象话,客户答不出也喂不进搜索)**:
- **这种人在哪、叫什么** — JD 不写目标公司,得你配:读 `_共享\公司档案.md` × 岗位性质,**网络检索+判断**出"对标/竞品/上下游/相邻行业藏这类人的公司"分类清单,带**职位名变体+锚点词**,交用户勾选。落 JD「开搜前三锚」块 + 喂第一轮强词。
- **标杆长什么样** — 用户有样板简历就放 `<岗位>\参考简历\`,**真读**(不读=没意义)抽正向画像/出身公司/技能锚点 → 落 JD 正向建议首行。没样板靠上一条兜。
- **什么看着像其实不是** — JD 只说谁合适;自然问"**哪些行业/公司/岗位,乍看像这份 JD、其实不对路?**"(走量客服像客服、镜头单件像整机结构)→ 落 JD 负向建议首行 + 排除词。

**intake 完成判据(逐项勾齐才回写 JD、进步骤0;别用"感觉够了"代替)**:① 硬约束行填满 ② 目标公司清单已交用户确认 ③ 样板已读或已标"无/欠" ④ 反向锚已问或已标"欠" ⑤ 用户说"就按这个搜"。
- 答不全很正常(真实录取线常要看到拒绝才说清)→ 有样板靠样板三角定位,答不出的标"第一轮校准时确认",照样开搜。
- **快速通道**:用户明说"立刻搜、先欠着" → 放行但**记一笔欠哪块**;第一轮当校准用(出批后按反馈补)。成熟 JD + 画像齐 + 用户直说搜 → 快速过一眼即可。

## 0. 排队 + 抢锁(锁的生命周期只此一处,别处只引用)
搜人先往 `_共享/搜索队列.md` 追一行 `待办`;当 worker:抢锁 → 挑最早 `待办` → 跑 → 标完成。
```bash
node SC/lock.mjs acquire <岗位>   # 本批第一轮前。退出码 1=别处在搜,排队;0=占到
node SC/lock.mjs renew   <岗位>   # 每轮回搜前续命,防长循环被误判过期
node SC/lock.mjs release          # 步骤4 出 PDF 交用户复核时释放(不是流程最后)
```
- **一把锁 = 一"批"**:`acquire`(本批首轮前)→ 每轮 `renew` → **步骤4 出 PDF 时 `release`**(复核是纯等待、不碰 liepin,把账号让给别的岗)→ 复核完回来搜下批再 `acquire`。别从头持到尾(饿死别的岗),也别每轮 acquire/release(和排队打架)。
- 多岗同招各自在队列排队;靠"批次边界释放锁"自然交错。

## 0.5 登录:自己探,别问用户登没登
**直接拿第一轮 search 当探针**:`search.mjs` 强制无头、不弹窗,跑得通=已登录,连着往下跑;只有脚本报未登录才登录。
- 脚本报未登录/被踢 → 运行(唯一允许有头的命令),让用户扫码:
  ```bash
  LIEPIN_HEADLESS=false liepin login    # PowerShell: $env:LIEPIN_HEADLESS='false'; liepin login
  ```
  跟用户说「我打开登录页了,扫一下二维码就行」;扫完 → **重跑刚才那条 search**。
- **开搜前一次性告诉预期**:「开始搜了,每轮约 10–20 分钟(慢是为了别被封),我每轮报你一句,凑够一批叫你看。」

## 1. 粗筛(脚本自动登 CSV)
```bash
node SC/search.mjs "<岗位>\去重台账.csv" "关键词" --city 深圳 --limit 40
```
一步做完:search → 去掉台账已召回的 → 新人登记 `未精筛` → 打印窄表(id/姓名/现职/年龄/公司/年限/期望薪资/学历/期望职位)。**登 CSV 是查重命根,已强制做**。
- **卡住保护**:search 6 分钟硬超时(resume 2.5 分钟),卡死会强杀报错、不静默挂死。报超时 → 告诉用户"搜索卡住已中断",**关残留 Chrome、确认已登录、过几分钟再试或临时 `--limit` 调小**,别立刻重刷。
- 卡片判不了能力(实做/真伪/方向细节要精筛),粗筛只分三桶:
  1. **卡面就能判死** → `粗筛不合适`:硬约束出局(城市/期望薪资/年限/学历/年龄)、期望职位明显转行、卡面可见的通用排除(见步骤2)。
  2. **~10 个最对口** → 拉进精筛(别只挑 5、也别宽到 20+)。
  3. **其余卡面过得去/拿不准** → 留 `未精筛`(**干净库存**:召回变稀时直接挑它再精筛,不重搜)。**拿不准就留未精筛,别误标粗筛不合适**。
  ```bash
  node SC/dedup.mjs set "<岗位>\去重台账.csv" 粗筛不合适 <卡面否的id...>
  ```
- `--limit 40` 起步(自动翻页,别 `--page`):漏斗要宽,~40-50 里挑 ~10。**别为快降到 20**。健康比 `未精筛:精筛 ≈ 4:1`,`dedup.mjs stats` 随时自查。

## 2. 精筛(批量取、按 JD 下判决)
拉粗筛挑的 ~10 份,**一个循环拉完 = 1 回合,别开一份判一份**:
```bash
node SC/fetch.mjs <id1> <id2> ...   # 逐份打窄字段,内置抖动
```
每份过三关(对照该岗 JD):
1. **硬约束&去向**:硬约束粗筛卡片已初筛;精筛复核 `want_city`(期望城市不在本岗城市=想迁走,慎留)。
2. **正/负向建议**:命中正向画像几条?命中负向排除信号 → 砍。
3. **真伪**:**实做主看 `work_history` 每段的 `duty`**(时间线下那段实际做了啥,精筛主料)——判成色/方向;`self_descr` 作补充(自述,空泛堆技能名词的警惕注水);再叠薪资-成色错配(带队多年要地板价=注水)、方向偏离 → 砍。段头时长/公司看稳定性(跳槽/空窗)。

**通用排除(岗位中立,叠在 JD 上,命中即砍)**:① 简历空洞看不出职责产出;② 频繁跳槽(多段 <1 年且无任何 ≥2 年稳定段);③ **在职+暂无跳槽打算**(`work_status`"暂不考虑/暂无跳槽打算"——约不动、议价难,统一砍;"离职找工作/在职-看机会"保留);④ **近一年无工作**(末段结束超 1 年)。①卡片看不出留精筛,②③④卡片能初判的粗筛就先排。

**归档**:过三关、值得给用户看 → `待定`;其余 → `精筛不合适`("够得着但不对"直接砍)。
```bash
node SC/dedup.mjs set "<岗位>\去重台账.csv" 待定     <过关id...>
node SC/dedup.mjs set "<岗位>\去重台账.csv" 精筛不合适 <淘汰id...>
```

## 3. 够一批没?
攒够 ~5 个 `待定`(或到目标人数)没?
- **没够** → 回步骤 1 搜下一页(别停问用户)。
- **够了** → 步骤 4。
- **方向熔断**:本次循环已粗筛 ~100 人(≈2 页)却 **0 进 `待定`** → **停、反馈用户**:搜了 N 人 0 命中,说清都倒在哪关/哪类,多半方向或标准有偏,等校准。别闷头刷。
- **路径枯竭**(召回越来越稀)→ 先挖 `未精筛` 库存;仍无 → 停,报情况 + 换路径建议。

## 4. 出 PDF + 交用户(不开网页)
这批 `待定` 一次性出 PDF 到 `待定/`。**默认不开 Chrome**(占 profile 挡搜索)。
```bash
LIEPIN_PDF_DIR="<岗位>\待定" node SC/pdf.mjs "<id1>=张三-大疆" "<id2>=李四-影石"
node SC/lock.mjs release    # 出完即释放,复核不碰 liepin,让账号给别的岗
```
交付消息**只两样**:①把 `pdf.mjs` 打印的每份路径列成**可点链接**(`[姓名·标签](<岗位>/待定/简历_<标签>.pdf)`,href 用正斜杠,用户点开不必翻文件夹);②**问为什么**(合适/不合适各带理由——下一轮 JD 收敛的唯一燃料,不能只收"行/不行"):
> 「这批 N 份出好了,直接点开看:
> 1. [张三 · 大疆](<岗位>/待定/简历_张三-大疆.pdf)
> 2. [李四 · 影石](<岗位>/待定/简历_李四-影石.pdf)
>
> 逐个看完跟我说哪些合适、哪些不合适,**各自为什么**:对的对在哪(背景/公司/作品/某段经历)、不对的差在哪(方向偏/成色虚/硬约束)。说得越细,下一轮越准。」
- 用户只甩"行/不行"没给理由 → **追问为什么**(步骤6 全靠它)。
- **`待定/` 留删**:入选的 PDF 留下(候选人池直链);落选的拍板后删;下批出新 PDF 前清掉上批未入选的。

## 5. 入池
用户复核拍板后:
- 合适 → `候选人池.md` 加一行(姓名·现职·经验·薪资·学历·一句判断 + `[PDF](待定/简历_<标签>.pdf)`、`[猎聘](https://lpt.liepin.com/cvview/showresumedetail?resIdEncode=<id>)`),`dedup.mjs set ... 入选`,PDF 留下。
- 不合适 → `dedup.mjs set ... 精筛不合适`、删 PDF。

## 6. 迭代:进化下一轮关键词(**本流程成败在此**)
**核心认知:真正改变候选池的只有搜索关键词。** 词决定捞来谁(池子),正/负向建议只决定怎么筛。**池子捞错,再严的筛也救不回。** 别"随便换个词再用更严标准卡一轮"——把上一轮结果当数据、把搜索词当假设来修正。

**A. 发动机:扒「入选」简历反推下轮的词**(逐个**真读**,别拍脑袋):
- **职位名变体** ← 他们简历上自称什么?同岗多种叫法都拿来当并列(OR)词。
- **出身公司+当时岗位** ← 来自哪些公司、在那做的什么?某段特别对口往往是**那家公司+当时那岗做的事**契合 → 开「目标公司」轴,**先顺藤摸瓜搜同公司同岗位的人**,再扩到其对标/竞品/上下游。
- **技术/工具/项目名词** ← 反复出现的工具/项目/产品词,都是锚点。

**B. 诊断本轮每个词、决定怎么调(一次只动一个轴,才看得出哪步起效):**

| 本轮表现 | 诊断 | 怎么调 |
|---|---|---|
| 召回多但大多不对口 | 太宽/有歧义 | 收紧:加硬约束、换更具体职位变体、加排除词 |
| 召回太少/快枯竭 | 太窄 | 放宽:同义近义词、职位变体,或扩邻近行业 |
| 连 ≥2 轮净入池 0 | 该词判死 | 移进弱搜索词弃用,换锚点 |

关键词轴轮换:① 职位名+变体 ② 目标公司(对标/竞品/上下游)③ 技术/工具名词 ④ 项目/产品名词 ⑤ 邻近行业(本行枯竭时)。**公司名别单独当搜索词**(召回该公司全职能员工、非本岗位)。

**C. 追加一行到 JD 表——用 `jd-round.mjs`,别手编表格**(手编漏个空行就把行甩出表格):
```bash
node SC/jd-round.mjs "<岗位>\JD.md" "R_（日期）·搜A/筛B/提报C" "<强搜索词>" "<弱搜索词>" "<正向建议>" "<负向建议>"
```
- **强** ← 本轮有效词 + A 步从入选简历新抽的词。**弱** ← 本轮无效/太宽/枯竭的词 + 为什么废。**正向** ← 用户说的"为什么合适" + 你真读入选的交叉印证。**负向** ← 用户毙掉的人归纳成排除信号。每格要点即可,别散文、别重抄硬约束。

**D. 回扫错杀(本轮标准/方向有调整时必做,否则越搜越少)**:先动手 `acquire` 重新占锁。按**新**标准复判已召回的库存,别只往前找新词:
- `dedup.mjs list <csv> 未精筛` → 库存里没细看的,直接精筛(本就是该挖的)。
- 标准明显变向时(某硬约束放宽/新增正向信号)→ `dedup.mjs list <csv> 精筛不合适` 取近几轮**可能受影响的**重新 fetch + 复判(要重拉简历、耗调用,只挑相关的、别全表重刷)。
- 翻案的 → `set ... 待定`,并入本批。**"越搜越少"多半就是只往前搜、不回头捡。**

**E.** 用**进化后的关键词**回步骤 0 继续,到满额收尾。
